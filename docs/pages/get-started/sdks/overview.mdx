ZeroDev ships SDKs for browser, mobile, and server-side stacks. Pick the one that matches your runtime.

## Client-side

For applications that run in a user's browser or on their device.

- **[TypeScript / JavaScript](/get-started/sdks/client-side/typescript)** — the core `@zerodev/sdk` package, for browser apps.
- **[React Hooks (waas, legacy)](/advanced/react-hooks/getting-started)** — `@zerodev/waas`. Maintenance mode.
- **[Embedded Wallet (React)](/smart-accounts/embedded-wallet/quickstart)** — `@zerodev/wallet-react` for embedded smart wallet flows.
- **[iOS (Swift)](/get-started/sdks/client-side/ios)** — Omni SDK Swift binding.
- **[Android (Kotlin)](/get-started/sdks/client-side/android)** — Omni SDK Kotlin binding.

## Server-side

For backend services, scripts, and agents.

- **[Node.js / TypeScript](/get-started/sdks/server-side/nodejs)** — `@zerodev/sdk` running server-side.
- **[Go](/advanced/go-sdk)** — legacy Go SDK.
- **[Python](/get-started/sdks/server-side/python)** — Omni SDK Python binding.
- **[Rust](/get-started/sdks/server-side/rust)** — Omni SDK Rust binding.
- **[C](/get-started/sdks/server-side/c)** — Omni SDK C FFI for cross-language embedding.
